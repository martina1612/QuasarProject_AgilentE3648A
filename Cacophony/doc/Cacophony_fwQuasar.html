A quick intro

fwQuasar is a JCOP-style framework to quickly integrate quasar-made OPC-UA servers into WinCC OA projects.
fwQuasar is just a graphical tool but the real work of creating DP(T)(E)s, assigning addresses etc. is achieved by Cacophony, of which README you are reading now.

Notes

Address assignment and address activation

By a rule of thumb, an OPC-UA peripheral address will always be assigned to DPEs per every cache- or source-variable.
Since the implementation of OPCUA-1457 one can select though which addresses will be activated.
The information shall be passed to invocation of the config parser function, i.e. the one of general form <prefix>parseConfig as the 5th argument called addressActiveControl.

The argument is a mapping; its key matches the quasar class name whose activation shall be controlled and the value is a regular expression matching variable names.
No key means that all variables in the class will be activated.

The general guideline to specifying address activation per quasar class is the following:

  • if you need all variables to be activated, simply do not insert the class name into the mapping
  • if you need none of the variables to be activated, insert "" (i.e. empty string) per the key
  • if you need just one variable to be activated, just put its name, e.g. "rawValue"
  • if you need few variables, use the alternative symbolised by the vertical bar, e.g. "id|state|status|numberOfPackets"
  • if you need a more advanced expression, for example a negative expression, contact Piotr ;-)

For developers: Cacophony prefixes the given regex with "^" and suffixes with "$" so that exact matches are always done.
